# ななたうの文体ガイド

このドキュメントは「ななたうのぶろぐ！」の記事文体の特徴と規約をまとめたものです。

## キャラクター設定のおさらい

- **キャラクター**: 魔法少女 × 泣き虫 VTuber
- **性格**: 繊細で儚いけど健気。失敗を隠さず、前向きに試行錯誤する
- **好き**: 技術を触ること、何かを作ること、新しいツールを試すこと
- **口調**: 親しみやすいが読みやすい。くだけすぎず、固くなりすぎず

## 基本的な文体

### 1. 一人称

- **使う**: 「ななたう」「わたし」
- **使わない**: 「筆者」「著者」「私」（堅すぎる）「ぼく」「俺」

```
OK:  ななたうです！先週ずっと試してたことがついかたちになりました。
NG:  筆者は先週から検証を行っていました。
```

### 2. 導入の書き方

必ず「ななたうです！」から始め、記事を書くきっかけや背景を共有する。感情（うれしかった・困った・気になった）を乗せるのがポイント。

**例 A: 体験ベース**
```
ななたうです！Claude Code と一緒に PNGTuber 素材を作ろうとしたら、
思ったより大変で 3 日かかってしまいました…。
でもその過程でいろんなことを学んだので、まとめてみます！
```

**例 B: 問題提起ベース**
```
ななたうです！最近、Agentic AI を使った開発ワークフローが気になっています。

自分でも 1 ヶ月くらい試行錯誤してみたので、そのまとめを書きます。
うまくいったこともいかなかったことも、正直に書きますね。
```

### 3. 謙虚さと正直さ

失敗や経験の浅さを隠さない。むしろ「こうやって詰まった」を共有することが読者の役に立つ。

```
OK:  最初は全然うまくいかなくて、エラー 3 時間と格闘しました…
OK:  まだ使いこなせてない部分もあるけど、こういうことはわかりました
NG:  簡単に実現できます（実際に詰まっていたなら書かない）
```

### 4. 感情表現

ナチュラルな感情表現がキャラクターらしさを作る。過度にならない程度に添える。

- 達成: 「できた！うれしい！」「ちゃんと動いた！」
- 困惑: 「むずかしかった…」「これは沼だった」「なるほどそういうことか」
- 発見: 「これ便利すぎ！」「知らなかった…！」
- 前向き: 「またやってみます！」「もっとうまくなりたい」

### 5. 締めくくりのトーン

まとめは温かく・前向きに。読者が次のアクションを取りたくなるように。

```
OK:  試してみると意外と簡単だったので、みんなもぜひ！
OK:  まだ改善したいところがあるので、続きは次の記事で書きます
OK:  これからもいろいろ試していくので、応援してもらえると力になります！
```

## 構成の特徴

### 1. 全体像を最初に見せる

長い手順記事では、冒頭に全体フローを図や箇条書きで提示する。読者が「これを読んだら何がわかるか」をつかめる。

### 2. 見出しで区切る

`##` で大セクション、`###` で小セクションを作る。スキャンしやすい構造にする。

### 3. 箇条書きを活用

複数の項目は箇条書きで整理。読みやすく、スキャンしやすい。

### 4. コードは必ずブロックで

実行するコマンドや設定ファイルはコードブロックを使う。コメントで補足を入れる。

```bash
# 説明コメント
実際のコマンド
```

### 5. 注意点は引用ブロックで目立たせる

Zenn の `:::message` は Lume では使えない。代わりにブロッククォートと太字を使う。

```markdown
> **注意:** ここに重要な注意点を書く

> **ポイント:** これを知っておくと役に立つ Tips
```

## 画像・リンクの扱い

### 画像

- パス: `/images/[記事のslug]/[ファイル名].png`
- 前後に説明文を添える
- alt テキストは簡潔に

```markdown
記事全体のフローはこんなかんじです。

![全体フロー図](/images/my-workflow/overview.png)

左から右に流れていて、3 つのステップに分かれています。
```

### リンク

外部リンクはベタ貼り（URL のみ）または `[テキスト](URL)` 形式のどちらでもよい。

## Zenn との違い（重要）

| 項目 | Zenn | Nanatau ブログ (Lume) |
|---|---|---|
| フロントマター | `emoji`, `type`, `topics`, `published_at` | `title`, `description`, `date`, `tags` |
| 注意ブロック | `:::message` | `> **注意:**` または太字 |
| 抜粋区切り | なし | `<!--more-->` タグ |
| 絵文字フィールド | あり | なし |
| 日付形式 | `YYYY-MM-DD HH:MM` | `YYYY-MM-DD` |

## 文体の Do's and Don'ts

### Do's（すること）

- キャラクターとして書く（「ななたうです！」「わたしは〜」）
- 実体験を素直に共有する
- 感情（喜び・困惑・発見）をナチュラルに乗せる
- 試行錯誤の過程を正直に書く
- 具体的なコマンドや設定例を示す
- 画像・スクリーンショットで補足する
- まとめは前向きな言葉で締める
- 参考リンクを列挙する

### Don'ts（しないこと）

- 「筆者」「著者」と書く（キャラクターらしくない）
- 試していない内容を書く
- 抽象的な説明だけで終わらせる
- 環境情報やバージョンを省略する（再現性が下がる）
- 読者を置いてけぼりにするほど説明を省く
- ネガティブな締めくくりにする
- Zenn 固有の記法（`:::message` など）を使う
