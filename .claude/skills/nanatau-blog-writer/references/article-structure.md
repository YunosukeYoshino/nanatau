# ななたうブログ 記事構成パターン

このドキュメントは「ななたうのぶろぐ！」の記事構成パターン 3 種をまとめたものです。

## パターン 1: 技術チュートリアル型

**用途**: 「○○ を作ってみた」「○○ をやってみた」「○○ で○○ する方法」
**例**: Claude Code で PNGTuber 素材を自動生成してみた / Lume でブログを作った

### 構成

1. 導入（ななたうです！ + きっかけ + 記事の概要）
2. 使ったツール・サービス（任意）
3. 全体像（図または箇条書きで手順を先出し）
4. 各ステップの詳細（Step 1 〜 Step N）
5. まとめ（やってわかったこと + 次にやりたいこと）
6. X フォロー誘導
7. 参考リンク

### テンプレート

```markdown
ななたうです！[きっかけ・何を試したか]。
[記事の予告：今回は○○について書きます]

<!--more-->

## 使ったもの（任意）

- ツール1: 説明
- ツール2: 説明

## 全体の流れ

![フロー図](/images/slug/overview.png)

以下の [N] ステップで進めます:

1. ステップ1: 説明
2. ステップ2: 説明
3. ステップ3: 説明

### Step 1. [ステップ名]

[概要と詳細説明]

```bash
コマンド例
```

[実行結果・注意点]

### Step 2. [ステップ名]

[同様に]

## まとめ

**今回は [主題] を紹介しました！**

1. わかったこと1
2. わかったこと2

[前向きな締め]

---

X でも発信しているので、フォローしてもらえるとうれしいです！

https://x.com/pomufgd

## 参考リンク

https://参照先
```

## パターン 2: ツール紹介・体験型

**用途**: 「○○ を使ってみた」「○○ の感想」「○○ vs ○○ を比べてみた」
**例**: Cursor を使ってみた感想 / Claude vs GPT-4 を試してみた

### 構成

1. 導入（ツールとの出会い・きっかけ）
2. ツール概要（何ができるか）
3. 使ってみた（機能別・シーン別に）
4. 良かった点・気になった点
5. まとめ（どんな人に向いているか + 個人的な感想）
6. X フォロー誘導
7. 参考リンク

### テンプレート

```markdown
ななたうです！最近 [ツール名] が気になっていたので、実際に使ってみました。

[ツールを知ったきっかけ・第一印象]

<!--more-->

https://ツールの公式サイト

## パターン 3: VTuber 活動・制作報告型

**用途**: キャラクター制作・配信準備・PNGTuber 素材・モーション・グッズなど VTuber 活動に関連するもの
**例**: PNGTuber 素材を Claude Code で作った話 / キャラクターデザインを決めるまでの話

### 構成

1. 導入（活動報告のきっかけ・何を作ったか）
2. 作ったものを見せる（スクリーンショット・画像・成果物）
3. 制作過程のポイント（工夫したこと・詰まったところ）
4. 振り返り（うまくいったこと・改善したいこと）
5. 次の展望（これからやりたいこと）
6. X フォロー誘導

### テンプレート

## セクション別の書き方ガイド

### 導入（共通）

**パターン A: 体験ベース**
```markdown
ななたうです！[具体的な体験。登壇・ハッカソン・制作など]。
[その体験から気づいたこと・記事を書こうと思ったきっかけ]
[経験の浅さや不完全さへの言及（任意）。正直に書くことで読者の信頼が生まれる]
```

**パターン B: 問題提起ベース**
```markdown
ななたうです！最近 [トピック] をずっと気にしていました。
[なぜ気になっていたか・背景]
[自分でどう取り組んだか + この記事で何を共有するか]
```

### 全体像の提示

```markdown
## [トピック] の全体像

![フロー図](/images/slug/overview.png)

今回の流れはこんなかんじです。以下の [N] ステップで構成されています:

1. ステップ1: 一言説明
2. ステップ2: 一言説明
3. ステップ3: 一言説明

それぞれくわしく紹介します！
```

### コマンド・コード例の提示

```markdown
[前後の説明文]

# [環境依存の部分があれば] はご自身の環境に合わせて変えてください
コマンド
```

[実行結果の説明・注意点があれば]
```

### 注意点・Tips

```markdown

## 記事執筆チェックリスト

- [ ] フロントマターに `title` `description` `date` `tags` が設定されている
- [ ] `description` が 80〜120 文字になっている
- [ ] 導入が「ななたうです！」で始まり、きっかけが書かれている
- [ ] `<!--more-->` が冒頭の自然な区切りに置かれている
- [ ] `draft: true` は公開前のみ付ける（公開時は削除）
- [ ] Zenn 固有の `:::message` を使っていない
- [ ] 全体像が図や箇条書きで提示されている（チュートリアル型）
- [ ] コマンド・コードがブロックで書かれている
- [ ] 画像パスが `/images/[slug]/[ファイル名].png` 形式になっている
- [ ] まとめで要点が再掲されている
- [ ] 締めが前向きな言葉で終わっている
- [ ] X フォロー誘導リンクがある
- [ ] 参考リンクが列挙されている
